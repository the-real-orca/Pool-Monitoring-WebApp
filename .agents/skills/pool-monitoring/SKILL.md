# Pool-Monitoring PWA — Project Skill

A Progressive Web App to monitor private swimming pools. Users record manual
measurements (temperature, pH, chlorine), log operational **events** (chemical
dosing, refill, backwash, winter), optionally extract pH/chlorine from a
photo of test strips via a multimodal AI, and view a **live dashboard**
(current values + 7-day trend chart) fed by MQTT sensor/pump data. A separate
service (**mqtt2mail**) emails periodic pool-status reports.

## Tech Stack (authoritative)

| Layer | Technology |
|-------|------------|
| Frontend | Vue 3 (Composition API, `<script setup>`, **JavaScript**), Vite 6, Tailwind v4 (CSS-first `@theme`), vite-plugin-pwa, uPlot (charts), Vitest |
| Backend | Python 3.12, FastAPI, Pydantic v2, paho-mqtt 2.x, official `openrouter` SDK, SQLite (WAL), pytest |
| Messaging | MQTT (Mosquitto), base-topic model |
| Reports | mqtt2mail (paho-mqtt + smtplib) |
| Infra | Docker Compose, Caddy (reverse proxy + Let's Encrypt), Alpine images |
| Deploy | vServer behind Caddy at `pool.io10.org`; `deploy-prepare.sh` builds a deploy package |

## Non-negotiable Architecture Principles

Read these before proposing any change (from `AGENTS.md`):

- **Flat & simple.** No unnecessary abstractions, dependencies, or file splits.
- **No Vue Router.** View state is `ref('live' | 'form' | 'event' | 'settings')` in `App.vue`.
- **No Pinia.** Module-level `reactive()` composables are the state layer.
- **No separate config module.** `os.getenv()` directly in `main.py`.
- **No Axios.** Native `fetch()` only, hardcoded relative `/api/*` paths (reverse-proxied).
- **v1 is otherwise stateless** apart from the Phase-20 SQLite live-data store.
- **Interaction language: German.** **Code comments: English.**

## Conventions & Workflow (must follow)

- **Tests are the quality gate.** Backend: `cd src/backend && pytest -v`
  (185 tests). Frontend: `cd src/frontend && npm run test` (94 tests).
  Publisher: `cd src/dev/mqtt-publisher && pytest -v` (11). mqtt2mail has **no
  tests yet** (known gap).
- **Lint:** Ruff (`line-length=100`, `select=["E","F","I"]`) for backend, ESLint for frontend.
- **Update `docs/plan.md` BEFORE committing** — it records what is *done*.
- **Git commits:** concise, imperative, English; append agent+model in brackets,
  e.g. `feat: add MQTT base-topic dispatch (Phase 23) [OpenCode / model]`.
  Commit only on explicit user request; check `git status` + `git diff` first.
- **Never commit secrets.** `src/.env`, `src/.env_production`, `deploy/.env`
  hold real credentials and are gitignored — keep it that way.
- **Always consult the TSD** (`docs/Pool-Monitoring - Technical Specification (TSD).md`)
  for the intended file layout and code patterns before writing code.

## Current Version & Key Facts (as of analysis)

- **App version `2.0`** in `src/backend/main.py` (`APP_VERSION`) and
  `SettingsPanel.vue`. NOTE: `frontend/package.json` still says `1.0.0` (bug).
- **Event API** (`POST /api/event`, topic `<base>/event`) — the old
  `/api/chem` was hard-cut in Phase 25 (404 regression test exists).
- **Base-topic MQTT model** (Phase 23): each `POOL_LIST[].topic` is a *base*;
  the backend subscribes `<base>/+` and dispatches by JSON payload content.
- **Pump field names are hard-coded constants** (`mainPump`, `solarPump`,
  `time`) — the old `LIVE_PUMP_FIELD_*` / `LIVE_TOPIC_*_TEMPLATE` env vars were
  removed.
